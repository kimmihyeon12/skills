# Sitemap 가이드

Sitemap은 User Story의 UI/화면 정보를 집약하여 **IA(Information Architecture) 원칙에 따라 설계한** 전체 화면 계층 구조 문서다.
단순한 화면 목록이 아니라, 사용자가 정보를 찾고 이동하는 구조 자체를 표현한다.

---

## 출력 경로

```
<워크스페이스>/sitemap.md
```

---

## 생성 시점

Step A-4에서 User Story 생성과 함께 자동 도출한다.
UI/화면 정보가 있는 Story가 하나 이상 존재할 때만 생성한다.

---

## 도출 규칙

1. 모든 User Story의 `UI/화면` 섹션을 스캔한다
2. 화면을 사용자 역할(페르소나)별로 그룹화한다. Product Brief의 대상 사용자 테이블을 기준으로 삼는다
3. MVP 포함 여부는 해당 화면을 요구하는 Story의 우선순위가 Must이면 `Y`, 그 외는 `N`
4. 관련 Story는 그 화면이 등장하는 US ID를 모두 열거한다 (복수 가능)
5. 화면 상태는 연결된 Story 중 가장 진행된 상태를 따른다

---

## IA 설계 원칙 적용

Sitemap을 생성할 때 단순 나열이 아닌 **구조적 설계**를 수행한다:

### 계층 구조 설계
- 화면을 의미 있는 섹션으로 그룹화한다 (5±2개 원칙)
- 상위 섹션은 사용자 언어로 명명한다 (내부 용어 지양)
- 핵심 태스크까지 3단계(depth) 이내로 도달 가능하도록 구성한다

### 레이블링
- 화면명은 사용자가 이해하는 언어로 작성한다
- 같은 개념은 동일한 이름을 사용한다 (일관성)
- 모호한 이름("기타", "서비스")은 사용하지 않는다

### 페르소나 분리
- 서로 다른 사용자 그룹의 진입점과 주요 경로를 명확히 분리한다
- 권한이 다른 사용자의 경로가 불필요하게 겹치지 않도록 한다

> **ia-guide 스킬 활용**: IA 설계에 깊이가 필요하다면 ia-guide 스킬을 함께 사용하면
> 네비게이션 계층 최적화, 레이블링 체계, 유저 플로우 설계를 전문가 관점으로 보완할 수 있다.

---

## 업데이트 시점

- Story가 추가·수정되고 UI/화면 정보가 변경될 때
- Story 상태가 변경될 때 (화면 상태 컬럼 동기화)
- Story가 cancelled될 때 해당 행의 상태를 `cancelled`로 갱신
