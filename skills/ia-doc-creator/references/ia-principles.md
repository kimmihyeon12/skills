# IA 핵심 원칙

IA 설계 시 반드시 적용할 원칙과 휴리스틱.

---

## 4가지 구성 요소 (Peter Morville)

### 1. Organization Systems (조직화)
정보를 어떻게 분류하고 그룹화할 것인가.

| 방식 | 설명 | 적합한 상황 |
|------|------|------------|
| 계층형 (Hierarchical) | 부모-자식 트리 구조 | 대부분의 웹/앱 |
| 순차형 (Sequential) | 단계별 흐름 | 온보딩, 체크아웃 |
| 매트릭스 (Matrix) | 다차원 분류 | 필터링이 중요한 카탈로그 |
| 태그/패싯 (Faceted) | 복수 속성 기반 필터 | 상품 검색, 콘텐츠 라이브러리 |

**그룹화 원칙:**
- 같은 그룹의 항목은 사용자 관점에서 연관성이 있어야 한다
- 그룹당 항목 수: 5±2개 권장 (Miller's Law)
- 상호 배타적(Mutually Exclusive), 전체 포괄(Collectively Exhaustive) — MECE 원칙

### 2. Labeling Systems (레이블링)
정보를 어떻게 표현하고 명명할 것인가.

**좋은 레이블의 조건:**
- 사용자의 언어를 사용한다 (내부 용어 X)
- 구체적이고 행동 지향적이다 ("설정" → "알림 설정")
- 일관성이 있다 — 같은 개념은 항상 같은 단어
- 간결하다 — 2~4 단어 이내

**레이블 안티패턴:**
- ❌ 모호함: "기타", "서비스", "솔루션"
- ❌ 내부 용어: "B2B 포털", "CRM 뷰"
- ❌ 불일치: "로그인" ↔ "사인인" 혼용

### 3. Navigation Systems (내비게이션)
사용자가 어떻게 정보를 탐색할 것인가.

| 내비게이션 유형 | 설명 | 예시 |
|----------------|------|------|
| 전역 (Global) | 모든 화면에서 접근 가능 | 상단 GNB, 하단 탭바 |
| 지역 (Local) | 현재 섹션 내 이동 | 사이드바, 탭 |
| 맥락적 (Contextual) | 콘텐츠 내 연관 링크 | 관련 상품, 다음 단계 |
| 보조 (Supplemental) | 빠른 접근 단축키 | 브레드크럼, 푸터 링크 |

**내비게이션 설계 원칙:**
- 현재 위치를 항상 표시한다 (브레드크럼 or 활성 메뉴 강조)
- 전역 내비게이션은 7개 이하 항목
- 모바일: 하단 탭바 5개 이하

### 4. Search Systems (검색)
사용자가 어떻게 원하는 것을 찾을 것인가.

- 콘텐츠가 많으면 검색 기능 필수
- 검색 결과 화면도 IA 설계 대상 (필터, 정렬, 빈 결과 처리)
- 자동완성, 오타 교정 여부 결정

## 핵심 설계 휴리스틱

### 3-Click Rule
핵심 콘텐츠 또는 태스크 완수까지 3번 이내 클릭으로 도달해야 한다.
- 클릭 수가 3을 초과하는 경로가 있으면 계층을 평탄화(flatten)하거나 단축 경로를 추가한다

### Progressive Disclosure
모든 정보를 한 번에 보여주지 않는다.
- 1단계: 핵심 정보만
- 2단계: 사용자가 원할 때 세부 정보 노출
- 예: 요약 카드 → 상세 페이지

### Scent of Information
사용자가 다음 단계로 가야 할 방향을 직감적으로 알 수 있어야 한다.
- 레이블, 아이콘, 배치가 목적지를 암시해야 한다
- "이게 여기 왜 있지?" 라는 질문이 나오면 scent 실패

### Entry Point Diversity
사용자는 항상 홈에서 시작하지 않는다.
- 검색 결과, 공유 링크, 북마크 등 어느 화면에서든 컨텍스트를 파악할 수 있어야 한다
- 각 화면은 독립적으로도 의미가 통해야 한다

## 페르소나별 설계 원칙

서로 다른 사용자 그룹은 서로 다른 정보 니즈와 멘탈 모델을 갖는다.

1. **역할별 분리**: 비회원 / 회원 / 관리자 등 역할에 따라 진입점과 경로를 분리한다
2. **과업 우선**: 페르소나의 주요 태스크가 가장 짧은 경로에 위치해야 한다
3. **교차 방지**: 권한이 다른 사용자의 경로가 불필요하게 겹치지 않도록 한다

## IA 감사 체크리스트

기존 구조를 검토할 때 사용한다:

- [ ] 모든 화면에 명확한 레이블이 있는가
- [ ] 같은 기능이 두 곳 이상에 중복 배치되지 않았는가
- [ ] 3-click 이내로 핵심 태스크 완수가 가능한가
- [ ] 전역 내비게이션 항목이 7개 이하인가
- [ ] 각 페르소나의 주요 경로가 최적화되어 있는가
- [ ] 레이블이 사용자 언어로 작성되어 있는가
- [ ] 화면 상태(빈 상태, 에러, 로딩)가 고려되었는가
- [ ] 모바일과 데스크톱 내비게이션 패턴이 적절히 분리되었는가
