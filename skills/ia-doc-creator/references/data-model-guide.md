# Data Model 가이드

Data Model은 User Story에서 도출된 엔티티와 그 관계를 정의하는 문서다. 백엔드 구현 전 데이터 구조 합의를 목적으로 한다.

---

## 출력 경로

```
<워크스페이스>/data-model.md
```

---

## 생성 시점

**MVP 프론트엔드 검토 통과 후** 생성한다. 이 조건을 충족하기 전에는 생성하지 않는다.
사용자가 명시적으로 요청하는 경우 예외적으로 생성할 수 있다.

---

## 도출 규칙

1. 모든 User Story와 Epic의 `데이터 모델` 섹션을 스캔한다
2. 엔티티를 통합하여 중복을 제거한다
3. 각 엔티티에 해당 엔티티가 언급된 Story를 링크한다
4. 관계는 Mermaid erDiagram으로 시각화한다
5. 필드 상세는 Story에서 명시된 속성만 기록한다. 추측하지 않는다

---

## 업데이트 시점

- Story 추가·수정으로 새 엔티티가 식별될 때
- MVP 범위가 변경될 때
- 백엔드 구현 중 스키마가 확정될 때

## 설계 아티팩트 위치 원칙

| 위치 | 내용 |
|------|------|
| Wiki (이 문서) | 무엇을 만들지 — 기획 컨텍스트, 엔티티 정의, 관계 |
| 소스코드 프로젝트 | 어떻게 만들지 — design-system.md, DB 스키마, API 명세 |

소스코드 프로젝트의 구현 산출물(디자인 시스템, ORM 모델 등)은 이 문서와 별도로 관리한다.
