# Product Brief 가이드

Product Brief는 프로젝트의 제품 컨텍스트를 캡처하는 문서다. 플랫폼 유형, 대상 사용자, 기술 제약 등 에이전트가 프로젝트를 수행할 때 필요한 기본 맥락을 제공한다.

---

## 출력 경로

```
<워크스페이스>/product-brief.md
```

---

## 템플릿

→ `templates/product-brief.md` 참고

---

## 작성 규칙

1. 입력에서 파악할 수 없는 항목은 `(미정)`으로 표기한다
2. **플랫폼 유형이 불명확하면 사용자에게 확인을 요청**한다 — 추측하지 않는다
3. 대상 사용자 테이블은 1~3행으로 간결하게 작성한다
4. 성공 지표 섹션은 입력에 관련 내용이 언급된 경우에만 포함한다. 없으면 섹션 자체를 생략한다
5. 제품 비전은 1-2문장으로 제한한다. 장황하게 쓰지 않는다
6. 기술 제약사항은 입력에 명시된 것만 기록한다. 없으면 `(미정)`
